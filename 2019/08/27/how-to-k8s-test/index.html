问题解决

目前常见的问题有三种：

不同Node之间的Pod网络不通，导致服务无法在注册中心（registry）注册，导致服务启动失败，无法提供服务

1. 不同Node之间的Pod网络不通解决方法

重启Node的Docker服务
使用命令, 查看POD所在的NODE上是不是所有的Pod都是READY 1/1
出现所有POD0/1 的时候需要重启当前节点的Docker服务

[解决方法]重启Docker
systemctl restart docker

参考一次Flannel和Docker网络不通定位问题

2. Pod本身存在配置问题，或者其他原因，导致服务启动失败，无法提供服务

3. Gateway网关不能访问

或者
